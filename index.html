方法论

静态站点友好，不把数据直接写死在 HTML 里

1. GitHub Pages 结构

页面入口使用 `index.html`，数据存放在 `data/latest.json`。这样上传到 GitHub Pages 后可以直接展示，不依赖本地脚本执行。

2. 手动更新按钮

按钮的作用是重新请求当前站点已发布的 JSON 文件，并刷新页面显示。它不会在浏览器里直接运行 Python 抓数脚本。

3. 真正生成新数据

如果要生成新的 `latest.json`，仍需本地运行更新脚本后再 push 到 GitHub。GitHub Pages 本身是静态托管，不能直接在网页按钮里执行抓取程序。
