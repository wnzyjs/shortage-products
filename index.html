方法论

静态页面负责展示，GitHub Actions 负责生成新数据

1. 页面按钮

“刷新已发布数据”只会重新加载当前站点上已经部署好的 `data/latest.js`。

2. 云端更新

真正生成新数据的是 GitHub Action `Update Dashboard Data`，它会运行 `scripts/update_dashboard.py` 并提交新文件。

3. 部署生效

新的数据提交后，GitHub Pages 会重新部署。部署完成前，网页刷新按钮看不到新变化。
